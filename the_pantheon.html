The Zoo Instructions

Dear CS majors, the Pantheon is now dead. You now have to use the Zoo.

We use SSH now instead of telnet. Welcome to the 21st century. Oh, you've never heard of SSH before? Well, you're in for a wild ride... SSHing into the Zoo computers that run Fedora Linux (IT'S NOT FUCKING UNIX. LINUX IS A UNIX-LIKE OPERATING SYSTEM. STOP PISSING OFF LINUS TORVALDS.) is like using your computer, but without the fancy pictures.. or any for that matter unless you count ASCII art.

To connect, open your favorite SSH client and connect to node.zoo.cs.yale.edu, which will connect you to a random available computer on the Zoo. Also configure your SSH client to use your netid as your username. And at least for the first login, use your net id password for password-based authentication.

Don't want to do the Duo authentication prompts every time? Setup ssh keys! Google how to generate an RSA key pair for your respective OS, then copy it in the right format to the ~/.ssh/authorized_keys file under your account on the Zoo. Now you will never have to type a password nor do two-factor authentication to log on to the Zoo again!

Vim Instructions

I'm also going to teach you the best text editor there is that you can use on the Zoo. Nothing even compares to the efficiency you can get using this editor. There are only a few essential things you need to know.

Actually, all you need to know is how to exit vim. Don't be one of those people that shutdown their computer because they open Vim, intentionally or not, and can not for the life of them figure out how to exit it. To exit, depending on your current mode within vim, press ESC once or twice. Then, type :q and hit enter. Congratulations, you have exited vim.

I'm not going to spoon feed you how to use every single piece of software on the Zoo. Use the man command to learn about them! For example to learn the basic of vim, run man vim.
